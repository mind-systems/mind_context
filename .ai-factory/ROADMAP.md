See `.ai-factory/notes/01-grpc-client-interceptor-flutter.md` for status-code mapping details.

- [x] **Create `src/grpc/grpc-auth.interceptor.ts`** ‚Äî NestJS `RpcInterceptor` that reads lowercase `authorization` from gRPC metadata, verifies JWT signature with `JwtService`, validates against `user_sessions` table via `SessionService.isValid()`, and stores the user payload in request context `[api]`
- [x] **Create `@GrpcCurrentUser()` decorator** ‚Äî reads user payload from gRPC execution context; place in `src/grpc/grpc-current-user.decorator.ts` `[api]`
- [x] **Apply interceptor** ‚Äî register `GrpcAuthInterceptor` globally in `AppModule` for the gRPC microservice, or per-controller using `@UseInterceptors` `[api]`

## Phase 3 ‚Äî Sync: Socket.io ‚Üí gRPC Streaming

### 3.1 Research spike ‚úì complete

> Design doc produced at `mind_api/docs/grpc/streaming-design.md`. All 7 questions answered in `.ai-factory/notes/04-grpc-streaming-design-full.md`. Decisions locked: `/live` ‚Üí bidi `LiveSession`, `/telemetry` ‚Üí bidi `StreamTelemetry`, `sync:changed` ‚Üí server-streaming `WatchChanges`. 3.2‚Äì3.6 are unblocked.

- [x] **Spike complete** ‚Äî streaming patterns decided, proto sketch produced, reconnect strategy defined, state-store migration path documented

### 3.2 Define streaming RPCs in proto

> Patterns decided: `/live` ‚Üí bidi, `/telemetry` ‚Üí bidi (per-batch acks are backpressure signals, not optional), `sync:changed` ‚Üí server-streaming. See `.ai-factory/notes/04-grpc-streaming-design-full.md` for full proto sketch.

- [x] **proto/sync.proto (streaming)** ‚Äî `WatchChanges(after_id?: int64) ‚Üí stream ChangeEvent(repeated SyncEventDto)` `[api]`
- [x] **proto/live.proto** ‚Äî `LiveSession(stream LiveRequest) ‚Üí stream LiveResponse`; `LiveRequest oneof`: `ActivityStartCmd(activity_type: ActivityType enum BREATH, ref_id?: string)`, `ActivityEndCmd`, `ActivityStopCmd`, `ActivityPauseCmd`, `ActivityResumeCmd`, `PresenceCmd(state: PresenceState enum FOREGROUND/BACKGROUND)`; `LiveResponse oneof`: `SessionStateEvent(live_session_id, status: SessionStatus enum ACTIVE/DISCONNECTED/COMPLETED/ABANDONED/INTERRUPTED/RESUMED, is_paused?: bool)`, `SessionErrorEvent(code, message, timestamp: int64)` `[api]`
- [x] **proto/telemetry.proto** ‚Äî `StreamTelemetry(stream TelemetryData) ‚Üí stream TelemetryResponse`; `TelemetryData`: session_id, timestamp: int64, module_id (string, e.g. "breath"), instruction_type (string, module-defined, e.g. "breath_phase"), data: google.protobuf.Struct (intentionally untyped); `TelemetryResponse oneof`: `TelemetryAck(session_id, received_count, dropped_count, max_samples_per_second, timestamp)`, `SessionErrorEvent` (imported from live.proto) `[api]`

### 3.3 Implement streaming gRPC services (mind_api)

- [x] **Verify `ChangeLogService.getChanges()`** ‚Äî confirm `getChanges(userId, after_id, limit)` exists and returns `{ events: ChangeRecord[], hasMore: boolean }`; if not, add the method to `src/sync/change-log.service.ts` before proceeding `[api]`
- [x] **Migrate `StateStore`** ‚Äî replace `socketMap: Map<string, AuthenticatedSocket>` with `streamMap: Map<string, ServerWritableStream<any, any>>`; update imports in `SyncNotifierService` and `LiveGateway` `[api]`
- [x] **Create `src/realtime/sync-stream.grpc.controller.ts` (replay phase)** ‚Äî `WatchChanges` server-streaming RPC: replay via `ChangeLogService.getChanges(userId, after_id, limit)` in a loop until `hasMore = false`, then enter live phase `[api]`
- [x] **Create `src/realtime/sync-stream.grpc.controller.ts` (live push phase)** ‚Äî register stream handle in `streamMap`; write `ChangeEvent` on `CHANGE_EVENT_LOGGED` event emitted by `SyncNotifierService` (the same event currently used for socket push); handle `fullResync` response from `SyncService` `[api]`
- [x] **Update `SyncNotifierService`** ‚Äî replace `socket.emit(SYNC_CHANGED, ...)` with `stream.write(changeEvent)` using `streamMap`; ensure `CHANGE_EVENT_LOGGED` is emitted via `EventEmitter2` or equivalent internal bus; debounce logic unchanged `[api]`

### 3.5 Replace LiveSocketService on mobile

> Reconnect strategy: `package:grpc` does not reconnect closed streams automatically. On `GrpcError` with retryable status, wait with exponential back-off and re-open the RPC call. On re-open, emit `connecting` ‚Üí `connected` state matching the existing state machine. `interceptStreaming` cannot be async ‚Äî token must be available synchronously (guaranteed if streaming is only opened after a successful unary call). See `.ai-factory/notes/04-grpc-streaming-design-full.md` (Flutter Client Implications section).

- [x] **Create `lib/Core/Grpc/LiveSessionGrpcService.dart`** ‚Äî holds `_liveCall` (bidi), `_telemetryCall` (bidi), `_syncCall` (server-streaming); implements reconnect with exponential back-off; maps domain events to `LiveRequest` proto messages; drives existing `BehaviorSubject<SocketConnectionState>` and stream controllers `[mobile]`

- [x] **Replace `SocketConnectionCoordinator.dart`** ‚Äî wire lifecycle (connect/disconnect/reconnect) to `LiveSessionGrpcService`; replace Socket.io connection state handling `[mobile]`
- [x] **Update `LiveSessionCoordinator.dart` in `packages/breath_module/`** ‚Äî replace calls to `ILiveSocketService` methods with equivalent gRPC stream sends if interface changes `[mobile]`

## Phase 6 ‚Äî Known Gaps (future work, not in current sprints)

### 6.1 Session revocation does not disconnect active streams

- [x] **Implement active-stream invalidation on logout** ‚Äî currently, `SessionService.revoke()` deletes the `user_sessions` row but any open gRPC stream on another device continues until the client disconnects. Fix requires tracking active streams per `userId` in memory and forcibly closing them when `revoke()` fires; in gRPC this means an in-memory store of open `ServerWritableStream` / bidi stream handles `[api]`

### 6.2 Rename streaming namespaces

- [x] **Rename `/live` ‚Üí `module_session`, `/telemetry` ‚Üí `module_stream`** ‚Äî current names are generic and don't communicate the modular architecture; `module_session` = session lifecycle control, `module_stream` = instruction log `[api]`
- [ ] **Define `module_biometrics` namespace** ‚Äî future channel for high-frequency body data (HR, EEG, accelerometer); separate from `module_stream` to isolate sensor flood from instruction events `[api]`
